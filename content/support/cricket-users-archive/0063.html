> I'm running AIX4.2.1 and perl 5.004. I was not able get Cricket to
> run successfully due to a problem getting the perl-shared portion
> of RRD installed correctly. So, I either need advice on making
> Cricket work with the RRD perl-piped code or a solution for making
> perl-shared work. There appear to be significant changes required

> I'm attempting to fix this problem on AIX with perl-shared. Suffice
> it so say, the problem with perl-shared boils down to this:
>
> When running the script in the perl-shared/examples directory, I get:
>
> $ ./shared-demo.pl
> ok 1 loading

> not ok 2 create

> the same error. I tried creating an rrd using rrdtool executable and
> that all seems to work okay.
>
> Thank you --Greg Redder
>

I have seen this same problem with AIX 4.2.1. I was never
able to solve it. I think it has to do with dynamically
loaded libraries not working. Are dynamic libraries
working in your installation of perl? In my case, when I
compiled perl and ran the tests, all the tests related to
dynamically loaded modules failed. You may want to try
statically linking all your modules. I upgraded my
machines to AIX 4.3.2 and used IBM's new VAC4.4 compiler
and perl works fine now (I have not tried RRD yet).
