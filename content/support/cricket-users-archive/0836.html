thanks! i had tried

        OID cpu1min 1.3.6.1.4.1.11.2.3.1.1.3.0

but on my system i must omit the trailing .0 -- sheesh (a thread on
cricket-users a few months ago clued me in to this). i actually got it
working last night.

my problem stemmed from my misunderstanding of the difference between
snmpwalk and snmpget (ucd-snmp). i can "walk" to the top of a table,
but i must "get" each element. so early on i had disregarded the OID
for loadTable.laLoad simply because i was unable to extract it with
snmpwalk.

the next confusion was that my cricket OID didn't work with the trailing
..0, though everything else does. i think ucd-snmp requires the trailing
..0 for scalars, but disallows it for table entries.

by the way, on my solaris machine the following OID doesn't seem to
work:

        OID freemem 1.3.6.1.4.1.11.2.3.1.1.7.0

though i am able to get some memory statistics from the ucdavis mib.

i wish i had more interesting stats than just cpu and memory usage.
going to fiddle around with netstats and iostats-type stuff now. will
post my config (of course) when i've got it the way i want it...

thanks again
