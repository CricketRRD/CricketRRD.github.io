Jeff, et al,

I had a problem running RRD 1.0.0 on AIX 4.3.2 with Cricket .69
whereby the code would not update any of its data files
and gave the error:

Cannot update
/home/cricket/cricket-config/../cricket-data//routers/router-name.rrd:
rrdopen can't open 'dummy'

I recently upgraded to rrd1.0.4 and cricket .70. I have the
same problem ;-( I was using a router subtree with one
router defined. I thought maybe that tree was corrupt, so I
decided to use the router-interfaces subtree with just two
interfaces. I get the same problem no matter what. I didn't
receive any response on this last plea so I'm hoping someone
has seen this behavior before and can help me narrow it down.
Here are more details:

Cricket's collector code calls:

RRDs::update($datafile, join(":", $when, @data2));

and gets the error:

Cannot update
/home/cricket/cricket-config/../cricket-data//routers/router-name.rrd:
rrdopen can't open 'dummy'

It's the "can't open 'dummy'" that is confusing me and preventing
things from working properly.

If I print out the variables passed to RRDs::update, I get:

$datafile =
/home/cricket/cricket-config/../cricket-data//routers/router-name.rrd

$when=N
@data2=1 1 28 36 3941364 10448088

These values all seem correct. Actually, I'm not quite sure what
the $when is doing, but ....

So, why is rrdupdate complaining about "dummy"? Any ideas, please?
