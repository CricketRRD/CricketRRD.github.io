Matthew Stier wrote:
> I've been trying to use "scale = %number%,/" in the graphing dictionary,
> but it appears that %tags% can only be used in the datasource
> dictionary.

Hmm. That's a really good idea... I think it'd be OK to simply expand
the graph dictionary with respect to the target dictionary.

The RIGHT solution is to change the expansion routines so that
%target::number% comes from the target dictionary. We could
specify which dictionary is the default, so that existing trees
wouldn't need to change at all (and so that the common case is
simple).

I'll think a little on this, and try to do something about it for
Cricket 0.65. It is too late for 0.64, which should be out later
today.
