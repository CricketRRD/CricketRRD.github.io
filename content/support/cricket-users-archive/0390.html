Jan -

Are you trying to monitor the ATM port of a router or an ATM switch?
If this is for a router, then the Ascend OID's won't work. The
listinterfaces script in Cricket will properly do the physical port
for an ATM interface by simply using the defaults for router ports.

ATM PVCs will not work this way and present a whole new can of worms.
Most versions of ATM PVC monitoring scripts for Cisco routers use a
telnet session to get the numbers and Cricket doesn't handle this type
of scripting very well - its a *LOT* of overhead. I finally resorted
to a custom script that updates using RRD directly for these
interfaces.
