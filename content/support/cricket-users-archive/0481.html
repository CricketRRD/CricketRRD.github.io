Not exactly, it pretty much goes like this.
Get new cricket up and running under cricket-0.70 directory, make sure to
test it so it is getting data, etc., etc.
While you are doing this, leave old cricket running.
When you are confident new one works, simply switch the cricket symlink you
are **supposed** to be using over to the new cricket directory.
The symlink was cricket -> cricket-0.63 and should now be cricket ->
cricket-0.70.

Good luck.
