From: "Adam B. Meltzer (KF6PDH)" <ameltzer@corp.webtv.net>

I'm playing with two totally seperate installations of Cricket right now and
having two totally wierd problems.

First off, with one instance it seems to be working just fine. I run collector,
and you see it getting valid stats and apparently writing to the RRD files.

[23-Sep-1999 09:56:42 ] Retrieving data (EXEC: /home/smrtgint/cricket-config/../util/netAppRate netapp-101) for netapp-101
[23-Sep-1999 09:56:42 ] Retrieved data for netapp-101 (): 90

But, if I look at the RRD file (even after running collector maybe 500 times
throughout several hours) here is what it will ALWAYS look like:

(strings ~/cricket-data/netapps/netapp-101.rrd):

0001
GAUGE
AVERAGE
AVERAGE
AVERAGE
AVERAGE
UNKN

Wierd, eh? If I nuke the old RRD files and try again it writes new ones but
same thing with no data - so it is seeing the right RRD files.

Next problem is totally unrelated. It's getting the stats, and the RRD files
look all good and all, but whenever I try to view the graph mini-grapher ALWAYS
fails with the 'Failed to draw graph' pic. I've checked things from the command
line and server-logs and can't find any errors.

Both are totally bone-stock Cricket installs with all the proper (and most
recent versions of) the modules.

Anyone seen such wierd behavior?
