Hi cricket-users,

I'm having a problem with grapher.cgi.

Sometimes the octets _hourly_ graph for an interface is not produced
correctly. Just a single ~325 bits/second average spike is drawn at the
current time. The "last update" averages are OK and the Octets Daily and
all UcastPackets graphs are correct.

The problem is not consistent, sometimes the correct hourly graph is
drawn. It seems to hit all interfaces at the same time. The best change of
a correct graph seems to be immediately after an RRD update.

I'm using the standard Defaults for the router interface subtree
and the target list was generated with listInterfaces, so it should be
pretty basic.

Any hints on where to look for the problem or maybe someone else has
experienced the same?

I'm using:
        cricket-0.63
        rrdtool-0.99.25
        FreeBSD 2.2.6-RELEASE

Thanks for any help.
