> Let's talk about this and see if we can make certain we are doing the
> least work to fix the problem you are having.

Fair enough. There are three typical problem graphs:

1) "Main page" graphs which can't afford the hit of CGIs at all. These
   could refer into the static graph area.

2) "Main page" graphs which are accessed frequently and should therefore
   not be generated each time. Caching works fine for these.

3) Complicated graphs which use mtargets-ops and take a lot of time to
   compute. Caching doesn't solve the problem since I don't want anyone
   to have to wait for them to be generated.
