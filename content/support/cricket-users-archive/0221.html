Question:

I have multiple identical routers that I need to monitor. I ran
listInterfaces on all the routers and gave each router it's own file, I
placed these files under the router-interfaces directory. On the generated
web page, I get all of the interfaces I wanted, however, the description is
all the same, this is because the target names are identical on all
different files since they are identical routers. Each file looks something
like this:

target --default--
        router = gwy.blah.blah

target fddi0_net0_FDDI
        interface-name = "fddi0/net0 FDDI"
        short-desc = "#1 900EF/MP Router"

target eth0_net1_Ethernet
        interface-name = "eth0/net1 Ethernet"
        short-desc = "#1 900EF/MP Router"

target eth1_net2_Ethernet
        interface-name = "eth1/net2 Ethernet"
        short-desc = "#1 900EF/MP Router"

target eth2_net3_Ethernet
        interface-name = "eth2/net3 Ethernet"
        short-desc = "#1 900EF/MP Router"

of course, the router= and short-desc is changed in every file.

How do I get it to uniquely monitor each interface, right now it gives
duplicate descriptions and brings you to the same page no matter which item
the group of interfaces you choose. For instance, there is 5 total routers,
which gives us 5 total fddi0 interfaces, 1 for each router. However, on web
page, clicking on any one of them brings you to only 1 page. This is
because target names are identical, however, you would think it would know
the difference since router= is changed in every file.

I tried changing target name and interface-name but it just messes up data
collecting.

Surely, cricket was meant to grab data from multiple peices of equipment
that are identical??? Please send any suggestions or sample-config

Thanks
