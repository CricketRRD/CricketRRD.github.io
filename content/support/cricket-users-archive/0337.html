From: Jeff Allen <jra@corp.webtv.net>

This version of Cricket is a bugfix version. If you upgraded to
Cricket 0.68, I'm sorry. This is the version you should have waited
for. Whoops.

This version fixes bugs in the new monitoring stuff (thanks Javier),
making it useful for the first time, we hope. (The bugs were
integration problems I made, which Javier politely fixed, with only
minor gnashing of teeth.)

Here's the CHANGES:

        New features, fixes in version 0.69 (16-Jul-99):

                A bunch of namespace fixes:
                The collector can again make directories when necessary.
                Instance mapping works. Lots of other fixes...

                Monitoring is now documented, and seems to work.
                Please read the description of the tag named
                'monitor-thresholds' and discuss this stuff on the
                mailing list. Javier Muniz, who wrote it, will be available
                to hel pus learn about it in the coming weeks.

                The new unknown-is-zero tag in the targets dictionary
                makes it possible to add together graphs which have unknown
                regions and not have the unknown regions blot out the entire
                graph.

I'll be down visiting Tobias Friday and busy this weekend. So it might
be a good idea to install this on a test system only, since I won't be
around this weekend to debug it.
