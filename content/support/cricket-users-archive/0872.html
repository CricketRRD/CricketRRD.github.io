From: Grimshaw Stuart <stuart.grimshaw@blackburn.gov.uk>

Having successfully configured cricket to monitor various things,
routers, printers etc, I've gathered my first bit of data over the past
week. However, I'm getting a problem displaying certain graphs...

I havn't altered the defaults file for the router-interfaces, and the
only thing contained in the interfaces file is the targets themselves,
(see below)

When I try to view a graph for one of the datasources, (the serial port)
I get these messages in the web servers error log:

[10-Jan-2000 11:02:56*] Actual graph width (595) differs from width-hint
(596).
[10-Jan-2000 11:02:56*] Actual graph height (255) differs from
height-hint (267

The other ports on the same device work fine, and I only get this when I
try and display the "Octets" view, the packets and errors view are fine.

I've tried turning off the graph hints in the defaults file now, but all
that changes is the errors dont appear in the log file :) I just get the
good old "Premature end etc etc"

This problem is hapenning on more than one, but not all of my
router-interfaces.

interfaces file :

target --default--
        router = 10.16.0.1

target 3Com_NETBuilderSER_1-1
        interface-name = 3Com_NETBuilderSER/1-1

target 3Com_NETBuilderETH_2-1
        interface-name = 3Com_NETBuilderETH/2-1
