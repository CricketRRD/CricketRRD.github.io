Ragnar Kjørstad wrote:
> The problem is that the disk-usage (and size) is reported i blocks. I
> would like to log the usage in bytes instead. I would prefer not having
> to update the configuration-files whenever blocksize is changed, so I
> would like cricket to log the product of StorageUsed and
> StorageAllocationUnits. How can I do that?

Cricket cannot do scaling operations on values fetched via SNMP before
inserting the value into RRD. The best way to do this is to use an
external program to fetch the two values and multiply them, then
return the value to Cricket. There are examples of programs that fetch
things via SNMP then post-processes the data in the util directory
(pmlines.pl and usrModemUsage).
