Ahh, I'm actually quite relieved to read that it's broken, and that I'm not
doing something wrong. I'm glad to read that you're releasing a more
soundly functioning version of Monitor.pm.

I do wonder if you're considering the addition of new types of monitor
actions. So, for a value monitor your might be able to write something
like:

threshold-monitors = ifInOctets:value:0:10:EXEC:/usr/bin/set:/usr/bin/clear

threshold-monitors =
ifInOctets:value:0:10:FUNC:print(\"WARNING!\"):print(\"It's OK\")

threshold-monitors = ifInOctets:value:0:10:SNMP

-or just use the normal notification method-

threshold-monitors = ifInOctets:value:0:10

Does this seem reasonable?
