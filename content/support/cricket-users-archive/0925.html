I'm new in cricket, but I thing there is a bug in grapher.cgi script.

grapher.cgi generate a .png file if passing arguments target, type, etc. and
dslist.

But it generates an empty graph if passing 'view' instead of 'dslist'
argument. In the same time everywhere in the scripts (grapher.cgi and
mini-graph.cgi) the only arguments passed and connected with ds name is
'view'.

DoHtml function in grapher.cgi deals with 'view' arguments correctly, but
doGraph function (which actually make the graph) doesn't deal with 'view'
argument. It looks only for 'dslist' argument, which is not passed (the
'view' argument is passing instead) and the graph produced is empty.

Is there a fix for this?
