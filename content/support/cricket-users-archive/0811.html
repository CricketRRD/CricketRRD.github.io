I have several graphs that, for historical reasons, should be unscaled.
The y-min and y-max options works fine for this purpose, however with
one disadvantage: the only work in the --default-- dictionary. According
to Matthew Stier in http://www.sugarat.net/hypermail/cricket/0134.html,
this is a bug in cricket-0.69, which according to my strained eyes is
fixed in cricket-0.70. However, it still doesn't work in the target
dictionary. This config gives a scaled graph:

target FastEthernet3_0_0
    interface-name = FastEthernet3/0/0
    short-desc = "Fast ethernet to Okern"
    y-max = 104857600
    y-min = 0
