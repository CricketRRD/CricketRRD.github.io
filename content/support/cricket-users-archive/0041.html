From: Noam Freedman <noam@noam.com>

Has anyone tried to make cricket generate some graphs from cron and
some graphs on the fly? I would be happy to modify the code to do this,
but I want to make sure I'm not recreating someones work.

In our situation, it makes a lot of sense to generate almost all of our
graphs dynamically. However we have a handful which would be better served
by periodic creation rather than dynamic creation.

I imagine this as a simple:

target blah-blah
        graph-generate = dynamic
or
        graph-generate = static
        graph-lifetime = 5

The grapher.cgi would know to first look for the static image in a set
location (~/cricket-graphs/....), and if it does not find it there it
would generate it on the fly.

Then a second cron job could be run in addition to the collector which
would create the static graphs as they are needed.

Since this is my first time posting, I would like to say how much I like
cricket. The code is very well written and easy to customize.

- Noam
