Hi Everyone,

I am using the latest release of Cricket and RRDTOOL
on a Sun Ultra 1 running fully patched Solaris 2.6.

Everything is working great at the moment, I have modified
'listInterfaces' to include a nice bit of html by default as
a long-descr, it pulls router name, interface type (has an
array of interface types), interface speed etc and includes
it on a zoomed view. If anyone's interested, mail me and you
can have it.

Anyway... I have got a multi-view working which is great,
but I am interested in two things now:

1. I need to have different size graphs for multiviews and
zoomed views. i.e. I would like small, perhaps 300x150 graphs
for multi-views and then revert to the 500x200 graphs when
zooming to them. I tried putting a new graphs --default--
tag in the config file (Peering_Points) and this over-rode
the main defaults at the start of the config tree. Any suggestions?

2. I need an aggregate view of some interfaces, I know that
mtargets should do it but it fails with a server error, the
config I have for it is as follows:

target Aggregate
        short-desc = "aggregate port usage"
        mtarget = "/router-interfaces/routera/ethernet5_0;
                                /router-interfaces/routerb/ethernet2_1;
                        /router-interfaces/routerc/ethernet0"

The same code works with the targets tag but not mtarget.

The apache server error is as follows:

[24-Aug-1999 17:12:41*] Variable expansion uses undefined variable:
router.
Can't call method "configHash" on an undefined value at
/**HIDDEN DIRECTORY**/cricket/lib/Common/Map.pm line 126.
[Tue Aug 24 17:12:41 1999] [error] [client 194.205.255.249] Premature
end of script headers: /www/cgi-bin/grapher.cgi

Any ideas would be much appreciated!

Thanks in advance,
