From: Jeff Allen <jra@corp.webtv.net>

Brad S. Dreisbach wrote:
> I had some email dialog with a list member this morning.
> Trying to figure out how to combine targets into an aggregate.

> He informed me that I should put
>
> mtargets-ops = "++++"

Close. That should be:

> mtargets-ops = "+,+,+,+"

What's really happening inside Cricket is that mtargets-ops is getting
added to the end of the list of datasources, which are getting
processed by RRD Tool as an RPN expression:

> [28-Apr-1999 13:05:31*] Unable to create graph: invalid rpn expression
> 'sds0,sds2,sds4,sds6,sds8,++++'

This is not documented very well since we are still experimenting with
it here. Sorry.

I'm thinking that for the common case of summing all the targets, we
should provide some syntactic sugar that turns 'mtargets-ops = "sum"'
into the right number of +'s automatically.
