From: Jeff Allen <jra@corp.webtv.net>

This version of Cricket has four simple fixes. From the CHANGES file:

        New features, fixes in version 0.67 (09-Jun-99):

                Fix: Added quoting to the inst tag to protect the
                argument of map() from eval.

                Fix: made overzealous warning from Perl
                (in Log.pm) go away.

                Fix: A vector instance with one element is now
                correctly treated like a fixed-instance scalar.
                If you didn't understand that, don't worry. What
                it means is that Cricket will Do the Right Thing.

                Fix: missed a conversion of expandString. Instance
                mapping now works again.

If you are running 0.66 now, you should upgrade to 0.67. It should be
a no-brainer upgrade, and you'll avoid some known bugs.

Now that Cricket is, ahem, stable again, expect to see some
interesting features in the coming weeks. To avoid angering the gods
of software development, I'm afraid I am going to have to leave it at
that... no details to announce yet. :)
