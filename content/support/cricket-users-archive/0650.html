Greetings,

I've successfully setup Cricket on a Linux box a few months ago, but due to
a job change, I'm now attempting the same thing on a NT box ( I know -
boo/hiss - I'm working on getting a Linux box).

Anyway, I'm running everything under Windows NT and IIS 4.0 and it's running
pretty good - with one exception (of course).

The graphs for Router, CPU, Memory, & Temp all work good (although the Avg,
and Max values are always 0 - the Current is good), but the graphs for the
router interfaces fail everytime (Failed to draw Graph, check your server
log for details). I thought it may have been a problem with the map
routine, so I created a Test-Interfaces directory and hard coded a couple
interfaces (no map(interface)), but that gives me the same results.

If anyone has any ideas, that would be greatly appreciated. I've started
looking at the exec function in mini-graph.cgi, but it works on the Router,
CPU, Temp, and Memory - so I'm not sure why it would react differently on
the other items.

Thanks in advance for any Help
