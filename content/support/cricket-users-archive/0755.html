Wow, I've never had someone have the same problem as me at the exact same
time, thanks! :) After reading the suExec web page I was able to get my
own simple cgi sripts to work, but not the linked scripts. My solution was
to just forget about running the scripts out of the cricket's home
directory and instead run them (via the standard links) in
/home/httpd/html/cricket instead. (Personally I've never liked having to
type in that tilde any ways. :) ) I'm pretty new to Linux, but it seems to
me that suExec isn't ran for any scripts that are run of the standard
/home/httpd directory. Maybe the beginner.txt file should be changed so
that this (http/html/cricket) is the standard install location for the web
files? It might help RedHat Linux beginners like me any ways.
