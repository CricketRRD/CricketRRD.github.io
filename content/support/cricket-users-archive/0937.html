        I've discarded the idea of using generate-statics to make static GIF files
for use in the "Description" field of a target. I've figured out that if I
remove the &rand=765 or whatever the number happens to be, from the URL to
an image, it gives me the current image. So I right-clicked and choose
"Copy Image Source" from my interface summary for my hourly graph, and
pasted that into my "short-desc" field in my target file, and remove the
&rand argument. Now I'm not wasting CPU time generating static images every
five minutes.
        This has worked for all my interfaces, and targets. However, I am getting
this error message an awful lot from most of my routers:

        [18-Jan-2000 18:53:53*] Variable expansion uses undefined variable: domain.

        I'm not entirely sure what's generating it, but I'm getting a similar one
from pasting my URLs to the images in there - it's trying to expand the %2F
in the URL to a variable. As long as you ignore it, it still works just
fine - all of my graphs are accurate, and showing up correctly. However, I'm
not sure where that error message above is coming from yet. It seems to be
yet another minor annoyance, but I thought I'd bring some stuff to light, to
the list.

        Another thing that we're doing with MRTG that Cricket wasn't doing by
default was that we had: <META HTTP-EQUIV="Refresh" CONTENT=300 > in our
head, so the page automatically refreshed. I added this myself with a
simple html dictionary, but for future reference, maybe something like this
should be put in by default, or suggested in a FAQ or something of the
sorts? Just trying to cater to the non-system engineers out there. (;
