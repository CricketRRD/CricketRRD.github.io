I must be missing something simple here but all the queries that cricket
generates return undefined.
for example when I query;

snmpwalk server community .1.3.6.1.4.1.1896.4.1
I get the expected result.
enterprises.1896.4.1.0 = 1

If I do an snmpget suchas:
snmpget server community .1.3.6.1.4.1.1896.4.1.0
I also get the expected result.
enterprises.1896.4.1.0 = 1

So what am I doing wrong ? How does cricket and / or the snmp.pm
determine which type of snmp query to use for a given OID (walk vs get)
? If I put a trailing .0 on the OID I'm querying it gets stripped off as
evidenced but the -loglevel debug output.
