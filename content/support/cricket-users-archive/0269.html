Alfred Nagl wrote:
> More generally asking, is it possible to first fetch a table (via snmp)
> and then use these received values further in fetching counters?

Not right now, but it will be soon. The new eval expansion syntax is
meant for this and other things. It will allow you to do an SNMP fetch
in the middle of target expansion, so you could set inst with the
result of an SNMP fetch.

It would look something like this:

                target foo
                                inst = "{snmpget('blah')}"
