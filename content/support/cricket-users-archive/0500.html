Folks,

        Anyone using named OIDs w/ cricket to grab information via SNMP? E.g.:

OID loadavg1m enterprises.ucdavis.loadTable.laEntry.loadaveLoad.1
OID loadavg5m enterprises.ucdavis.loadTable.laEntry.loadaveLoad.2
OID loadavg15m enterprises.ucdavis.loadTable.laEntry.loadaveLoad.3
OID diskpct1 enterprises.ucdavis.diskTable.dskEntry.diskPercent.1
OID diskpct2 enterprises.ucdavis.diskTable.dskEntry.diskPercent.2
OID diskpct3 enterprises.ucdavis.diskTable.dskEntry.diskPercent.3

        These are documented in UCD SNMP, and work with their version of
snmpget and snmpwalk, but I only see numeric OIDs in the default
configuration files I can find for cricket.

        If you can use only numeric OIDs, does anyone have any idea what
these would convert to in numeric form?

        Thanks!
