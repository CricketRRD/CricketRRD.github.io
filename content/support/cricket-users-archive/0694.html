From: "Berg, Ivan" <iberg@montana.edu>

This is because the interfaces are named the same and cricket stores
data files based on the target name which is the same on both routers.
It is giving a error because you get data, then attempt to overwrite the
data right after with more data from the other router.

To solve this:
Create two seperate directory's under cricket-config for each router.
Then place the Targets file for each router under their own directories.
Then cricket will create unique data files.
