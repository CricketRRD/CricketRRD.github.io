Hi George,

I just went through this here at my company. The solution I came up with is as follows:

Use the "generate-static = true" in the Defaults files where you want to display to customers
and setup the cron job to run the generate-statics program (I run it every hour).

I then setup a separate link in apache (http://www.yourserver.com/customer/) that points to the
point in the config tree you want them to have access to
(/home/cricket/public_html/static/router-interfaces/customer/) and put in a .htaccess file in
this directory that has the customers (and admins) username/password. This got me to the point
where I could see raw directories and individual gifs.

Final set was to whip up some quick perl that generated index.html's in each directory that
have the same "look and feel" as my normal cricket pages and have links appropriate to the
content of that directory :-) ie if the top dir has 3 sub dir's it has links to the
index.html's in those subdir's auto-named according the sub-dir name. The final step was to
generate an index at the target level with links to all the appropriate targets in the final
directory and individual html files for each target.

<pauses to catch breath>

If you or anyone else on the list is interested the quick and dirty perl scripts I used to do
this, lemme know and I'll put them up on my FTP server :-)

The documentation for generating statics says "Making HTML pages that incorporate the generated
image files is left as an exercise to the reader." heh heh heh, so if you want to save some
time I have done this exercise and can share :-)
