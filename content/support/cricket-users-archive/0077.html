From: <roofpig@the.satanic.org>

        It seems that it would be quite useful to have
targetType inheritance, where one targetType includes
the definitions of a "parent" targetType, and then adds
addional data-sources or views.

        A good example of this is sample-config/routers/Defaults.
Most of the data-sources are the same for all the types of Cisco
routers defined. Rather than repeating these for each type, a parent
type of "Cisco" could be defined, which includes all the common
data-sources (cpu, memory, etc.). Then sub-types, like "Cisco-2500,
Cisco-7500" could be defined which inherit all the definitions from
"Cisco" and then add the platform specific definitions.

        This occured to me while writing a config-tree for APC
UPS's. Some of mine have external temperature sensors (with
associated MIB entries), some do not.
